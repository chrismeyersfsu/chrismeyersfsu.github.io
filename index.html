lttng - C, Low-Overhead User/Kernel Logging

Used low overhead logging library (<1%) lttng-ust to log data traces that could later be used to verify the corretness of my system. Observed that the logged output was garbled. Constructed a minimal, concise, multi-threaded test case that was reliable in triggering the garbled log. With this test case, the author of the code was able to find and fix the problem.

gevent - Python, IPC Web (nginx) <-> Python Scripts (environment)

For performance reasons I used a unix socket as the IPC between gevent and nginx instead of the default network socket. This resulted in a logging failure when trying to query the remote host name of the socket. I recomoneded a quick fix to this solution that would return UNIX_SOCKET for the hostname when the hostname could not be determined.
